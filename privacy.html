This policy explains how ADACT SOLUTIONS PTY LTD (ABN 62 700 080 155, ACN 700 080 155) handles information received through the ADACT website, contact forms and ADAC Viewer.

Information collected

Depending on how you use the website, ADACT may receive:

  • website and device information supplied through analytics services;
  • contact details and messages that you voluntarily submit;
  • an anonymous browser identifier used to associate permitted ADAC Viewer activity; and
  • ADAC project usage information described below when an XML is loaded into the configured viewer.

ADAC Viewer project usage information

XML and DXF file contents, filenames, individual asset attributes and edited values are processed in the browser and are not uploaded to ADACT as part of project usage tracking.

When project usage tracking is enabled, ADACT privately records relevant project and professional metadata, source software, approximate drawing location, ADAC file characteristics and controlled names of viewer tools used. Sample XML files supplied with the viewer are excluded.

The surveyor and engineer named in an XML file may not be the person using the viewer. Unless a user separately signs in, viewer activity is associated with an anonymous browser identifier rather than a verified individual.

How information is used

ADACT uses collected information to:

  • understand demand for ADAC services and viewer features;
  • improve compatibility, usability and product decisions;
  • respond to enquiries and suggestions;
  • protect the website and investigate misuse; and
  • meet legal and business record-keeping obligations.

ADACT does not use XML project metadata to change, certify or submit the user's source XML.

Service providers and network requests

ADACT uses service providers including Google Analytics, Firebase, Web3Forms, map tile providers and public ArcGIS REST services. These services may process technical request information under their own terms and privacy practices.

Displaying maps, imagery, boundaries or asset overlays may send the currently viewed geographic extent to the relevant mapping provider. Project metadata collected through the viewer is kept separate from Google Analytics.

The project usage database is configured for storage in Sydney, Australia. Some service providers may also process technical information in other countries where they or their suppliers operate, including the United States.

Storage, access and security

ADAC Viewer project records are stored in a restricted database. Public viewer users can create permitted records but cannot read the project register. Access is limited to authorised ADACT administrators.

ADACT takes reasonable steps to protect collected information and reviews it periodically. Information is retained only while it remains reasonably required for the stated purposes, legal obligations or legitimate business records.

Your choices

The configured viewer records the project usage information described above automatically. Do not load an XML through the public viewer if you are not authorised to share its declared project metadata with ADACT.

You may ask whether ADACT holds information about you or a project, request correction, or ask that information be deleted where appropriate. ADACT may need sufficient details to locate and verify the relevant record.

Contact and complaints

Questions, access or correction requests, and privacy complaints can be sent to projects@adact.com.au. ADACT will review the request and respond within a reasonable period.
